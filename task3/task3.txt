Глобально проблемы тормозов сервера можно разделить на две:
•	Связанные с вводом выводом
•	Не связанные с вводом выводом

Первое чем нужно на мой взгляд заняться это пониманием того, на чём работает указанный сервер и выяснить следующее:
•	Есть ли на сервере контейнеризация? Если да, то какая?
•	Какой web server используется? Nginx, traeffic, Apache? – тут может крыться потенциальная проблема медленной работы, какая то ошибка или долгое ожидание а limit времени на request стоит очень большой.
•	Какие вообще демоны на сервере запущены? Можно посмотреть через утилиту top (htop вроде как нам не разрешен) ? Там может съедать процессорный ресурс посторонний процесс, начиная криво развёрнутым демоном, заканчивая майнером. Может наоборот малое количество workers для WSGI application может давать медленную работу
•	Далее нужно понять какие базы данных есть на сервере SQL, NOSql, брокеры сообщений, и соответственно посмотреть в репозиториях проектов обращение к ним (SQL запросы в основном) по различным причинам база может быть плохо настроена или сконфигурирована, кто-то мог некорректно написать Select вкупе с отсутствием нормального ограничения по таймауту можно ждать ответ очень долго = тормоза
